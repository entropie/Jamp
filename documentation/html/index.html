Config settings

$cfg['serverroot']

Default: 'none'
The address of the script. (like: http://www.ackro.de/Jamp)

$cfg['streamroot']

Default: 'stream'
The directory where we store our cached files (XML, HTML.inc). The Apache user must have write rights.
Don't create this folder, the script does that.

$cfg['httpstreamroot']

Default: '$cfg['serverroot'] + $cfg[\streamroot']'
The directory where the cached files are stored, too. But this one must be the HTTP address. (like: http://www.ackro.de/Jamp/stream).
This one has to match with '$cfg['streamroot'].
Usually you have not to edit this one.

$cfg['dir_content']

Default: './content'
The directory where we store the symlinks to our songfolders and files.

$cfg['command_string']

Default: '!'
A string which is used to add commands in the 'command line'. If this string is the first char, the command will be evaluated als command, if not we search the database.

$cfg['userLogin']

Default: '0'
If is set to '1' visiters have to log in, bevor they can do anything else.

$cfg['adminIps']

Default: 'none'
Set a bounce of IP's if $cfg['userLogin'] is set to '0'. Each user with this ip has admin rights.
Example:
    $cfg['adminIps'][] = "192.168.42.1";
    $cfg['adminIps'][] = "192.168.42.2";

$cfg['m3ufile']

Default: 'a random string'
Just a random string.

$cfg['sort_updatedb']

Default: '1'
If is set to '1' the files will be "natural" sorted bevor we write them into the database

$cfg['mp3_dirs']

Default: 'none'
For each folder 'updatedb' starts a new instance, and searches the folder recursiv for music-files.
Example:
    $cfg['mp3_dirs'][] = '/home/ftp/music';
    $cfg['mp3_dirs'][] = '/usr/local/music';

$cfg['cookie_lifetime']

Default: '(3600*24*30*12)'
The time in seconds the cookie lifes (3600*24*30*12 = 1 Jear).

$cfg['mp3blaster_plist_dir']

Default: '/etc/playlists'
The folder where playlists for offline using are stored, I use this with lirc.

$cfg['showShoutBox']

Default: '1'
If is set to '1' the Shoutbox is visible.

$cfg['sbLimit']

Default: '15'
The number of Shoutbox entrys which are visible.

$cfg['sbMaxTextLength']

Default: '255'
If a user writes more letters than this number, the text will be cutted and we show a link to the entire text.

$cfg['allowAnonymousXmlDump']

Default: '1'
If is set to '1' all users can do an xml dump (without authentication). If you put this script on public webserver turn this off. I use thhis for daily backups with wget.

$cfg['style']

Default: './templates/GreenWar'
The full path to the template.

$cfg['langFolder']

Default: './cfg/language'
The full path to the language folder.

$cfg['language']

Default: 'en'
The language you want to use.

$cfg['showusermenu']

Default: '1'
if is set to '1' users see her navigation (playlists etc). Strogly recomend.

$cfg['max_plist_height']

Default: '15'
The height of the playlist <select>. On 800x600 15 is the right number.

$cfg['mysql']

An array which contains MySql settings.

$cfg['logFolder']

Default: 'logs/'
The log folder.

$cfg['max_log_size']

Default: '1024*20'
Maximum size bevore a logfile is renamed/deleted.

$cfg['maxDevLogSize']

Default: '1024*1'
Maximum size bevore a DEVlogfile is renamed/deleted. (much less, performance issue)

$cfg['saveOldDevLogs']

Default: '1'
If is set to '1' old logfiles will be renamed if the max size is reached.

$cfg['showDevLog']

Default: '1'
If is set to '1' the devolog is visible for admin users.

$cfg['devLogLines']

Default: '23'
The numbers of devlog-lines which are visible.

$cfg['log']

The names of all logfiles. Without *.log
